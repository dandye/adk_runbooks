How A2A Communication Works

  1. Agent Registration: Each A2A agent exposes REST endpoints and agent cards
  2. Host Discovery: SOC Manager discovers available agents at startup
  3. Task Delegation: Manager sends structured messages to appropriate agents
  4. Form Interactions: Agents return interactive forms for data collection
  5. Response Processing: Manager synthesizes results and provides comprehensive analysis

ğŸ”§ Troubleshooting

Common Issues

Agent Connection Errors

Issue: "Failed to connect to agent at localhost:8001"
Solution: Ensure A2A agents are running before starting the host agent. Check ports are not in use.

Import Errors

Issue: "ModuleNotFoundError: No module named 'a2a_sdk'"
Solution: Activate virtual environment and run pip install -r requirements.txt

API Key Issues

Issue: "Invalid API key for Google Gemini"
Solution: Verify GOOGLE_API_KEY in manager/.env file and ensure API access is enabled.

Debug Mode

# Enable debug logging export ADK_LOG_LEVEL=DEBUG # Run agents with verbose output python run_server.py --debug

Port Configuration

If default ports (8001, 8002) are in use, update the configurations:

  • Modify port numbers in run_server.py files
  • Update agent URLs in host_agent.py
  • Adjust startup script port references
